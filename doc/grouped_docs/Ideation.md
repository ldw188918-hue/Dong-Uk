# 아이디어 노트 (Ideation): BizInsight Local

이 문서는 PRD를 바탕으로 구체적인 구현 방안, 추가 기능 아이디어, 그리고 사용자 경험(UX) 향상을 위한 브레인스토밍 결과를 정리한 것입니다.

## 1. 데이터 수집 및 전처리 고도화 (Data Ingestion & Preprocessing)

### 스마트 컬럼 매핑 (Smart Column Mapping)
*   **문제**: 사용자가 업로드하는 엑셀 파일마다 컬럼명이 다를 수 있음 (예: '날짜', '주문일', '결제일시').
*   **아이디어**: 
    *   **퍼지 매칭(Fuzzy Matching)**: 유사한 단어를 자동으로 찾아 매핑 제안.
    *   **사용자 수동 매핑 UI**: AI가 확신하지 못하는 경우, 드롭다운 메뉴를 통해 사용자가 직접 컬럼을 지정하도록 유도.
    *   **컬럼 템플릿 제공**: 자주 사용되는 POS 시스템(예: 배달의민족, 포스뱅크 등)별 표준 템플릿 제공.

### 데이터 정제 (Data Cleaning)
*   **이상치 탐지**: 터무니없이 높은 금액이나 미래 날짜 등 이상 데이터를 시각적으로 강조하여 사용자에게 확인 요청.
*   **결측치 처리 옵션**: 단순히 제외하는 것 외에 '0으로 채우기' 또는 '이전 값으로 채우기' 등 옵션 제공.

## 2. 시각화 및 대시보드 (Visualization & Dashboard)

### 스토리텔링형 대시보드 (Storytelling Dashboard)
*   단순한 차트 나열이 아닌, **"현황 -> 원인 -> 대책"** 흐름으로 배치.
*   예: [매출 추세 차트] 바로 옆에 [주요 등락 원인 AI 요약] 배치.

### 인터랙티브 차트 (Interactive Charts)
*   **Drill-down**: 월별 매출 그래프를 클릭하면 해당 월의 일별 매출로, 일별 매출을 클릭하면 시간대별 매출로 상세 진입.
*   **Brush & Filter**: 그래프의 특정 구간을 드래그하면, 다른 모든 그래프도 해당 기간으로 필터링되어 연동.

### 업종별 맞춤 뷰 (Industry-specific Views)
*   초기 설정에서 '업종(카페, 식당, 소매점)'을 선택하게 하여, 대시보드 구성을 최적화.
    *   **카페**: 시간대별/날씨별 판매량, 테이크아웃 비율 중요.
    *   **식당**: 테이블 회전율(데이터가 있다면), 메뉴별 마진율 중요.

## 3. AI 기반 인사이트 강화 (AI Insights)

### 프롬프트 엔지니어링 전략 (Prompt Engineering)
*   **페르소나 부여**: "당신은 20년 경력의 데이터 분석가이자 친절한 소상공인 컨설턴트입니다."
*   **Context 주입**: 단순히 데이터 통계만 주는 것이 아니라, "오늘은 비가 오는 금요일입니다"와 같은 외부 변수(가능하다면)를 함께 고려하도록 유도.

### 구체적인 액션 아이템 예시
*   **재고 관리**: "지난 3주간 '아메리카노' 원두 소진 속도가 10% 빨라졌습니다. 다음 발주 시 2kg 추가 주문을 고려하세요."
*   **프로모션**: "화요일 오후 2~4시가 가장 한가합니다. '해피 아워 20% 할인' 쿠폰을 발행해보는 것은 어떨까요?"

### 인사이트 신뢰도 (Confidence Score)
*   AI가 생성한 조언의 근거가 되는 데이터 수치를 함께 명시. (예: "매출이 15% 상승할 여지가 있습니다 (근거: 유사한 주말 패턴 분석 결과)")

## 4. UI/UX 디자인 (UI/UX Design)

*   **Minimality**: 복잡한 버튼을 숨기고, '파일 업로드' -> '분석 시작'이라는 단순한 플로우 강조.
*   **Loading Experience**: 분석이(특히 로컬 LLM 구동 시) 오래 걸릴 수 있으므로, "데이터를 읽고 있습니다...", "AI가 고민 중입니다..."와 같은 친근한 진행 상황 메시지 표시.
*   **Dark/Light Mode**: 야간에 매장을 마감하며 사용하는 사장님들을 위한 다크 모드 지원.

## 5. 기술적 고려사항 (Technical Considerations)

### 로컬 LLM 최적화
*   **모델 선택**: 사양 낮은 PC에서도 돌아가도록 Quantized된 모델(4bit 등) 사용 권장 (예: Llama 3 8B 4bit).
*   **Ollama 설치 감지**: Ollama가 설치되어 있지 않거나 실행 중이지 않을 때, 친절한 안내 메시지 및 설치 가이드 링크 제공.

### 배포 및 실행 (Deployment)
*   **단일 실행 파일**: Python을 모르는 사용자를 위해 PyInstaller 등을 이용해 `.exe` 파일 하나로 패키징.
*   **자동 업데이트**: 새로운 기능 추가 시 쉽게 업데이트할 수 있는 체계(또는 안내) 마련.

## 6. 향후 확장 아이디어 (Future Roadmap)

*   **PDF 리포트 내보내기**: 은행 대출 제줄용이나 파트너 공유용으로 깔끔하게 포맷팅된 PDF 다운로드 기능.
*   **다국어 지원**: 외국인 사장님들을 위한 영어/다국어 UI 지원.
*   **간편 장부 기능**: 비용(재료비, 인건비) 데이터도 입력받아 단순 매출이 아닌 '순수익' 분석으로 확장.
